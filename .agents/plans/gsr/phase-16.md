# GSR Phase 16: Web/Admin Transport Layers

## Summary

All 8 checklist items pass. The web and admin transport layers demonstrate clean separation of concerns (handlers → app → gateway → gRPC), consistent error handling, enforced module boundaries via architecture tests, and security-conscious session management. HTMX integration is consistent.

## Findings

### F16.1: Web Handlers Call gRPC, Not Storage — Compliant

**Severity:** style (no action needed)

Gateway pattern enforces service boundary. Each module implements a `gateway/` subpackage wrapping gRPC clients. Handlers call app services which call gateways. No direct storage imports.

### F16.2: Module Composition — Well-Documented & Tested

**Severity:** style (no action needed)

Architecture docs cover web layering model. Architecture tests enforce patterns: `TestFeatureModulesDoNotImportSiblingModules`, `TestModulesMountDoNotReadGatewayClientsFromDependencies`, `TestFeatureModulesFollowTemplate`. Composition injectable and testable.

### F16.3: HTTP Error Handling — Consistent

**Severity:** style (no action needed)

Shared `httperrors` package with typed Kinds. gRPC error mapping via `MapGRPCTransportError()`. Web-specific customization (`FailedPrecondition` → 409 Conflict). Admin adds `LogError()` for structured logging.

### F16.4: Template Rendering — Separated

**Severity:** style (no action needed)

Handlers build data structs, delegate rendering to `pagerender` or `htmx.RenderPage()`. Localization injected as parameter. HTMX branching centralized (fragment vs full-page).

### F16.5: Session Management — Secure

**Severity:** style (no action needed)

`sessioncookie` package: HttpOnly=true, SameSite=Lax, Secure conditional on HTTPS, explicit MaxAge=-1 for clear. Admin auth via OAuth token introspection with explicit `Active` check.

### F16.6: HTMX Integration — Consistent

**Severity:** style (no action needed)

Shared detection via `IsHTMXRequest()`. Centralized branching in `htmx.RenderPage()`. Header preservation (Set-Cookie). Title injection for HTMX page updates.

### F16.7: Admin Authorization — Well-Enforced

**Severity:** style (no action needed)

`requireAuth()` middleware protects `/app/*` routes. Token introspection, user ID context injection, downstream gRPC services enforce domain-level authz.

### F16.8: Static Asset Versioning — Embedded

**Severity:** style (no action needed)

Go `embed.FS` bundles assets into binary. ETags auto-generated by `http.FileServer`. No explicit cache-control headers needed.
